# Code2MD Go项目实现计划

## 1. 项目架构设计

```
code2md/
├── cmd/
│   └── main.go                    # 程序入口
├── internal/
│   ├── parser/                    # 代码解析器模块
│   │   ├── parser.go             # 解析器接口定义
│   │   ├── factory.go            # 解析器工厂
│   │   ├── go_parser.go          # Go语言解析器
│   │   ├── js_parser.go          # JavaScript/TypeScript解析器
│   │   ├── java_parser.go        # Java解析器
│   │   ├── python_parser.go      # Python解析器
│   │   ├── cpp_parser.go         # C++解析器
│   │   └── php_parser.go         # PHP解析器
│   ├── analyzer/                  # 代码分析器
│   │   └── analyzer.go           # 调用关系和依赖分析
│   ├── generator/                 # Markdown生成器
│   │   └── generator.go          # 文档生成逻辑
│   └── utils/                     # 工具函数
│       └── file.go               # 文件遍历和工具函数
├── models/                        # 数据模型
│   └── code_element.go           # 代码元素模型
├── config/                        # 配置
│   └── config.go                 # 命令行配置
├── go.mod
├── go.sum
├── Dockerfile
└── README.md
```

## 2. 核心技术方案

### 2.1 语言解析策略
- **Go**: 使用`go/ast`和`go/parser`（标准库）
- **JavaScript/TypeScript**: 使用`tree-sitter`配合对应语法树
- **Java**: 使用`tree-sitter-java`语法树
- **Python**: 使用`tree-sitter-python`或内置ast
- **C++**: 使用`tree-sitter-cpp`
- **PHP**: 使用`tree-sitter-php`

### 2.2 统一的解析器接口
```go
type Parser interface {
    Parse(filePath string) (*CodeInfo, error)
    GetLanguage() string
    SupportedExtensions() []string
}
```

## 3. 数据模型设计

### CodeElement (代码元素)
- **名称**: 元素名称
- **类型**: 函数/变量/类/接口/结构体
- **语言**: 所属语言
- **文件路径**: 所在文件
- **行号范围**: StartLine, EndLine
- **可见性**: public/private/protected
- **描述**: 功能描述
- **参数列表**: 函数参数
- **返回值**: 返回类型
- **调用关系**: 被哪些函数调用、调用哪些函数
- **引用关系**: 被哪些变量引用

## 4. 功能模块

### 4.1 命令行参数
- `-p, --path`: 项目路径（默认./）
- `-o, --output`: 输出文件名（默认code.md）
- `-l, --languages`: 指定语言（默认全部）
- `-v, --verbose`: 详细输出
- `--exclude`: 排除模式（如vendor, node_modules）

### 4.2 文件遍历
- 递归遍历目录
- 根据扩展名匹配解析器
- 过滤隐藏文件和排除目录

### 4.3 文档生成
- 项目结构概览
- 目录树结构
- 语言统计
- 代码元素清单（按类型分组）
- 调用关系图
- 完整代码（带行号和链接）

## 5. 实现步骤

1. **项目初始化**

   - 添加依赖（tree-sitter相关）

2. **核心解析器实现**
   - 实现解析器接口

3. **分析器**
   - 构建调用关系图

4. **生成器**

5. **主程序**
   - 参数解析
   - 协调各模块工作

6. **测试**
   - 单元测试
   - 集成测试

7. **Docker化**
   - 编写Dockerfile
   - 推送到Docker Hub
